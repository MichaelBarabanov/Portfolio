ComingSoon

Coming-Soon-Modus für Shopware 6 – pro Sales Channel oder einzelne URLs, mit Countdown, geheimem Preview-Link und sauberem SEO-Verhalten (HTTP 503 statt 200).

GitHub ansehen →

// features

Was es kann.

  • Pro Sales Channel aktivierbar oder auf einzelne URLs mit Wildcard begrenzt
  • Vollständig konfigurierbares Design – Farben, Logo, Text
  • Countdown mit automatischer Freischaltung zum Zieldatum
  • IP-Whitelist und geheimer Preview-Link (?michaPreview=TOKEN) mit 7-Tage-Cookie
  • SEO-sicher: HTTP 503 + Retry-After-Header + Noindex, kein Content-Leak an Suchmaschinen
  • Kein Build-Schritt nötig, cache-aware für Reverse-Proxies

// installation

Installation.

Plugin nach custom/plugins/ kopieren, dann installieren und aktivieren:

michael@shop:~
$ cp -r shopware-coming-soon custom/plugins/MichaComingSoon
$ bin/console plugin:refresh
$ bin/console plugin:install --activate MichaComingSoon
$ bin/console cache:clear
// häufige fragen

Häufige Fragen.

Warum HTTP 503 statt 200?
200 signalisiert Suchmaschinen "diese Seite ist der echte, dauerhafte Inhalt" – das führt dazu, dass die Coming-Soon-Seite selbst indexiert wird. 503 mit Retry-After sagt Crawlern "komm später wieder", ohne den echten Content preiszugeben oder das Ranking zu gefährden.
Was ist mit Reverse-Proxy- oder HTTP-Caches?
Die Antwort wird mit Headern ausgeliefert, die ein Zwischenspeichern der 503-Seite verhindern, damit nach der Freischaltung sofort der echte Shop ausgeliefert wird und nicht noch eine gecachte Coming-Soon-Seite.
Funktioniert das Admin-Backend weiterhin normal?
Ja – der Coming-Soon-Modus greift ausschließlich im Storefront. Das Admin bleibt uneingeschränkt erreichbar, damit während der Vorbereitungsphase normal weitergearbeitet werden kann.
