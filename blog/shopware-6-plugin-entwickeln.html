Shopware 6 Plugin entwickeln: Schritt-fur-Schritt von Null zum ersten Release

2026-06-10 · 12 min Lesezeit

Shopware 6 basiert auf Symfony und nutzt dessen Dependency-Injection-Container, Event-System und Console-Commands. Wer Symfony kennt, findet sich schnell zurecht. Dieses Tutorial fuhrt von der leeren Verzeichnisstruktur bis zum deploybaren Plugin.

1. Verzeichnisstruktur anlegen

Ein Shopware-6-Plugin liegt unter custom/plugins/MeinPlugin/. Die Mindeststruktur:

custom/plugins/MeinPlugin/
├── composer.json
├── src/
│   └── MeinPlugin.php
└── Resources/
    └── config/
        └── services.xml

Die composer.json definiert den Plugin-Typ und den Autoload-Pfad:

{
  "name": "vendor/mein-plugin",
  "type": "shopware-platform-plugin",
  "autoload": {
    "psr-4": {
      "MeinPlugin\\": "src/"
    }
  },
  "extra": {
    "shopware-plugin-class": "MeinPlugin\\MeinPlugin",
    "label": { "de-DE": "Mein Plugin", "en-GB": "My Plugin" }
  }
}

2. Plugin-Basisklasse

Die Hauptklasse erweitert Shopware\Core\Framework\Plugin und ist der Einstiegspunkt fur Installation, Update und Deinstallation:

<?php declare(strict_types=1);

namespace MeinPlugin;

use Shopware\Core\Framework\Plugin;
use Shopware\Core\Framework\Plugin\Context\InstallContext;

class MeinPlugin extends Plugin
{
    public function install(InstallContext $installContext): void
    {
        parent::install($installContext);
        // Migrationen, Custom Fields etc.
    }
}

3. Services per Dependency Injection registrieren

Services werden in Resources/config/services.xml registriert:

<?xml version="1.0" ?>
<container xmlns="http://symfony.com/schema/dic/services">
  <services>
    <service id="MeinPlugin\Service\MeinService" public="false">
      <argument type="service" id="product.repository" />
      <tag name="shopware.feature" flag="v6.5.0.0" />
    </service>
  </services>
</container>

4. Events subscriben

Shopware dispatcht hunderte von Events. Ein Subscriber registriert sich auf beliebige davon:

<?php declare(strict_types=1);

namespace MeinPlugin\Subscriber;

use Shopware\Core\Content\Product\Events\ProductListingCriteriaEvent;
use Symfony\Component\EventDispatcher\EventSubscriberInterface;

class ProductSubscriber implements EventSubscriberInterface
{
    public static function getSubscribedEvents(): array
    {
        return [
            ProductListingCriteriaEvent::class => 'onProductListing',
        ];
    }

    public function onProductListing(ProductListingCriteriaEvent $event): void
    {
        $event->getCriteria()->addAssociation('manufacturer');
    }
}

Den Subscriber in services.xml mit dem Tag kernel.event_subscriber registrieren:

<service id="MeinPlugin\Subscriber\ProductSubscriber">
  <tag name="kernel.event_subscriber" />
</service>

5. Admin-UI mit Vue.js erweitern

Das Shopware-Admin-Panel basiert auf Vue.js. Eigene Komponenten liegen unter Resources/app/administration/:

Resources/app/administration/
└── src/
    └── module/
        └── mein-modul/
            ├── index.js
            └── page/
                └── mein-modul-list/
                    ├── index.js
                    └── mein-modul-list.html.twig

Nach Anderungen im Admin-Teil muss das Theme neu gebaut werden:

bin/console plugin:refresh
bin/console plugin:install --activate MeinPlugin
bin/console theme:compile

6. Deployment via GitHub Actions

Ein simpler Workflow der bei jedem Push auf main das Plugin auf dem Server deployt:

name: Deploy Plugin
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy via rsync
        run: |
          rsync -az --delete ./src/ \
            ${{ "{{" }} secrets.SSH_USER }}@${{ "{{" }} secrets.SSH_HOST }}:\
            /var/www/shopware/custom/plugins/MeinPlugin/src/
      - name: Clear cache
        run: |
          ssh ${{ "{{" }} secrets.SSH_USER }}@${{ "{{" }} secrets.SSH_HOST }} \
            "cd /var/www/shopware && bin/console cache:clear"
SSH-Credentials als GitHub Secrets hinterlegen, nie im Repository committen.

Fazit

Der schnellste Weg zum ersten lauffahigen Plugin: composer.json und Plugin-Klasse anlegen, Plugin installieren, dann inkrementell Services und Subscriber hinzufugen. Das Symfony-Fundament macht Shopware-Plugins sehr testbar -- PHPUnit-Tests fur Services schreiben sich genau wie in jedem anderen Symfony-Projekt.
